# Conveyor Monitoring Dashboard

## Описание проекта

Conveyor Monitoring Dashboard — это frontend-приложение для мониторинга конвейерных линий, предназначенное для отображения состояния линий, работы с инцидентами и просмотра отчётных данных по сменам.

Проект реализован как одностраничное веб-приложение (SPA) и моделирует работу интерфейса оператора или диспетчера, который отслеживает производственные линии, получает информацию о проблемных событиях, работает с инцидентами и анализирует сводную статистику.

## Цель проекта

Цель проекта — разработать удобный и наглядный интерфейс мониторинга, который позволяет:
- просматривать текущее состояние конвейерных линий;
- видеть проблемные события и инциденты;
- открывать карточку отдельного инцидента;
- изменять статус инцидента;
- просматривать отчёты по дате и смене;
- использовать mock API для имитации backend-части;
- демонстрировать базовую realtime-логику и уведомления.

## Реализованный функционал

### 1. Overview / обзор линий
На главной странице отображаются конвейерные линии с основными параметрами:
- статус линии;
- скорость;
- нагрузка;
- количество камер;
- количество тревог за час.

Также реализованы:
- фильтрация линий по статусам;
- визуальное выделение тревожных линий;
- отображение счётчика уведомлений;
- базовые toast-уведомления о событиях.

### 2. Incidents / список инцидентов
На странице инцидентов выводится список событий с основной информацией:
- идентификатор инцидента;
- связанная линия;
- статус;
- ответственный;
- комментарий;
- дата создания.

Из списка можно перейти в карточку отдельного инцидента.

### 3. Incident / карточка инцидента
Для каждого инцидента реализована отдельная страница, на которой доступны:
- подробная информация по инциденту;
- история изменения статусов;
- форма изменения статуса;
- валидация формы;
- mock PATCH-запрос для обновления данных;
- базовый блок для изображения и bbox-рамки.

### 4. Report / отчёты
На странице отчётов реализованы:
- выбор даты;
- выбор смены;
- загрузка отчёта через mock API;
- отображение ключевых показателей;
- график событий по часам;
- график событий по линиям;
- экспорт данных отчёта в CSV.

### 5. Mock API
Для имитации backend-части в проекте используется MSW (Mock Service Worker).  
Через mock API обрабатываются запросы для:
- получения линий;
- получения списка инцидентов;
- получения карточки отдельного инцидента;
- изменения статуса инцидента;
- получения отчётов.

### 6. Realtime-логика
В проекте реализована базовая имитация realtime-событий:
- изменение состояния линий;
- генерация событий;
- отображение уведомлений в интерфейсе.

## Архитектурная идея проекта
Проект построен как frontend-приложение с mock backend-слоем.  
Интерфейс не привязан напрямую к локальным данным, а работает через запросы к mock API, что делает структуру приложения ближе к реальному production-подходу и упрощает дальнейший переход к настоящему backend.

## Текущий этап проекта
На данный момент проект находится на этапе рабочего MVP.  
Основные страницы, пользовательские сценарии и базовая логика реализованы.  
Проект можно запускать, демонстрировать и использовать как законченную учебную frontend-работу.

## Возможности дальнейшего развития
В дальнейшем проект можно доработать в следующих направлениях:
- подключение реального backend;
- расширение realtime-логики;
- улучшение UX и обработки ошибок;
- добавление тестов;
- дополнительная оптимизация и рефакторинг.
