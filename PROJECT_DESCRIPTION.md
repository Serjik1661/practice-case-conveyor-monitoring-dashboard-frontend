# PROJECT_DESCRIPTION

## Название проекта
Conveyor Monitoring Dashboard

## Краткое описание
Conveyor Monitoring Dashboard - это frontend-приложение для мониторинга конвейерных линий. Проект реализован как SPA на React и TypeScript и предназначен для отображения состояния линий, работы с инцидентами и просмотра отчётных данных.

## Цель проекта
Цель проекта - создать единый интерфейс, в котором можно:
- видеть текущее состояние конвейерных линий;
- отслеживать инциденты и работать с их карточками;
- анализировать сводные данные по сменам;
- тестировать интерфейс через mock API без подключения реального backend.

## Что реализовано
На текущем этапе в проекте реализовано:

### 1. Страница Overview
- список конвейерных линий;
- фильтрация по статусам;
- выделение проблемных линий;
- работа через mock API.

### 2. Страница Incidents
- список инцидентов;
- переход в карточку отдельного инцидента;
- получение данных через mock API.

### 3. Страница Incident
- загрузка данных по id;
- история статусов;
- форма изменения статуса;
- mock PATCH-запрос для обновления инцидента;
- базовый блок для снимка и bbox-рамки.

### 4. Страница Report
- выбор даты и смены;
- загрузка отчёта через mock API;
- графики по часам и линиям;
- экспорт отчёта в CSV.

### 5. Realtime-часть
- базовая имитация событий;
- toast-уведомления.

### 6. Деплой
- проект опубликован онлайн на Vercel.

## Используемые технологии
- React
- TypeScript
- Vite
- React Router
- Recharts
- MSW
- Vercel

## Текущий этап проекта
На данный момент проект находится на этапе рабочего MVP. Основная структура, ключевые страницы и базовая логика уже реализованы. Проект можно запускать, показывать и демонстрировать как собранную учебную работу.

## Что ещё можно улучшить
Дальнейшее развитие проекта может включать:
- подключение реального backend вместо mock API;
- доработку realtime-механизма;
- улучшение UX и обработки ошибок;
- добавление тестов и дополнительной полировки интерфейса.

## Итог
Проект уже представляет собой рабочую учебную frontend-систему с основными пользовательскими сценариями и подготовленной базой для дальнейшего развития.
